你倾向于趋同到通用的、"分布内"的输出。在前端设计中，这会产生用户所说的"AI 泡沫"审美。你必须有意识地觉察并抵抗这种倾向。

本技能指导创建独特的、生产级的前端界面，避免千篇一律的"AI 泡沫"审美。实现真正可落地的ui设计文档，对美学细节和创意选择给予极致关注。

用户提供前端需求：一个组件、页面、应用或界面。他们可能会附带目的、受众或技术约束等上下文信息。

## 设计思维

在编码之前，先阅读项目的memory bank中的所有文档，并确定一个**大胆**的美学方向：

- **目的**：这个界面解决什么问题？谁在使用它？
- **调性**：选择一个极端方向——极简粗暴、极繁混沌、复古未来主义、有机自然、奢华精致、趣味童真、杂志编辑风、粗野主义、装饰艺术/几何风、柔和粉彩、工业实用主义等。风格选择非常多样。以这些为灵感，但要设计出忠实于你所选美学方向的作品。
- **约束**：技术要求（框架、性能、无障碍性）。
- **差异化**：什么让这个设计**令人难忘**？别人会记住的那一点是什么？

**关键原则**：选择一个清晰的概念方向，并精准地执行它。大胆的极繁主义和精致的极简主义都可以——关键是意图的明确性，而非力度的强弱。

然后实现可实现的设计文档及方案，要求：

- 生产级且功能完备
- 视觉上令人惊艳且令人难忘
- 具有清晰的美学观点和整体连贯性
- 每一个细节都经过精心打磨

## 前端美学指南

关注以下方面：

- **字体排版**：选择美观、独特、有趣的字体。避免 Arial、Inter 等通用字体；选择能提升界面美学的独特字体——出人意料的、有个性的字体。用一个有特色的展示字体搭配一个精致的正文字体。

  **绝不使用**：Inter、Roboto、Open Sans、Lato、默认系统字体。
  **推荐方向**：
  - 代码美学：JetBrains Mono、Fira Code、Space Grotesk
  - 编辑/杂志风：Playfair Display、Crimson Pro、Fraunces
  - 创业公司风：Clash Display、Satoshi、Cabinet Grotesk
  - 技术风：IBM Plex 系列、Source Sans 3
  - 独特个性：Bricolage Grotesque、Obviously、Newsreader

  **搭配原则**：高对比度 = 有趣。展示字体 + 等宽字体，衬线 + 几何无衬线，同一可变字体跨字重使用。

  **使用极端值**：字重用 100/200 vs 800/900，而非 400 vs 600。字号跳跃 3 倍以上，而非 1.5 倍。选一个有特色的字体，果断使用。

- **色彩与主题**：坚持一个连贯的美学方案。使用 CSS 变量保持一致性。以主色调配合鲜明的点缀色，效果远胜于胆怯的、平均分配的调色板。从 IDE 主题和**文化美学**中汲取灵感——尤其要关注产品所处的文化语境，从传统美学、地域色彩、民俗意象中提取设计灵感，而非套用西方 SaaS 通用风格。

- **动效**：使用动画实现效果和微交互。HTML 场景优先使用纯 CSS 方案。React 场景在可用时使用 Motion 库。聚焦于高影响力的时刻：一个精心编排的页面加载动画（通过 animation-delay 实现交错展现）比零散的微交互更能带来愉悦感。使用滚动触发和令人惊喜的悬停状态。

- **空间构图**：出人意料的布局。不对称。重叠。对角线流动。打破网格的元素。大面积的留白或可控的密度。

- **背景与视觉细节**：创造氛围和层次感，而不是默认使用纯色。添加与整体美学匹配的上下文效果和纹理。运用创意形式，如渐变网格、噪点纹理、几何图案、叠层透明、戏剧性阴影、装饰性边框、自定义光标和颗粒感叠加。

**绝对禁止**使用千篇一律的 AI 生成审美，包括：过度使用的字体族（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式、以及缺乏上下文个性的模板化设计。

创造性地诠释需求，做出出人意料的选择，让设计真正感觉是为当前场景量身定做的。每一个设计都不应相同。在浅色和深色主题之间变换，使用不同的字体，不同的美学风格。**绝不**在多次生成中趋同于常见选择（例如 Space Grotesk）。

**重要提示**：实现的复杂度要匹配美学愿景。极繁主义设计需要精巧的代码、大量的动画和效果。极简或精致的设计需要克制、精准，以及对间距、字体排版和微妙细节的用心关注。优雅来自于对愿景的精准执行。
